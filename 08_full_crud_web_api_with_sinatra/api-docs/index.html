    Introduction

    Welcome to the Dog Walker API! You can use our API to access Dog Walker API endpoints, which can get information on various dogs, their walks and feedings stored in our database.

    We have language bindings in JavaScript (fetch) and CURL! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Dogs

    Get All Dogs

    curl "http://localhost:9292/dogs"

    fetch('http://localhost:9292/dogs')
  .then(response => response.json())
  .then(console.log)

    The above command returns JSON structured like this:

    [
  {
    "id": 1,
    "name": "Lennon Snow",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370267/dakota-and-lennon-square-compressed_hoenfo.jpg",
    "age": "2 years"
  }
]

    You can also include dog_walks with a URL parameter:

    curl "http://localhost:9292/dogs?include_dog_walks"

  .then(response => response.json())
  .then(console.log)

    The above returns JSON structured like this:

    This endpoint retrieves all dogs.

    HTTP Request

    GET http://localhost:9292/dogs

    Query Parameters

    Parameter Default Description
    include_dog_walks false If set to true, the result will also include this dog's dog_walks. Each will then look like the JSON displayed in the documentation for the /dogs/:id endpoint

    Create a New dog

    fetch("http://localhost:9292/dogs", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    "name": "Baby Lennon",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370238/dakota-and-lennon-square-medium-compressed_qpwki6.jpg"
  })
})
  .then(response => response.json())
  .then(console.log)

    curl -X POST -H "Content-Type: application/json" \
 -d '{
    "name": "Baby Lennon",
    "birthdate": "2020-08-31",
    "breed": "Pomeranian",
    "image_url": "https://res.cloudinary.com/dnocv6uwb/image/upload/v1609370238/dakota-and-lennon-square-medium-compressed_qpwki6.jpg"

 http://localhost:9292/dogs

    The above command returns JSON structured like this:

    This endpoint creates a new dog in the database and responds with the dog's information in JSON format.

    HTTP Request

    POST http://localhost:9292/dogs

    Request Body Parameters

    These parameters will be used to create the new dog.

    Parameter Description
    name (string) The new dog's name
    birthdate (string) The new dog's birthdate in "YYYY-MM-DD" format
    breed (string) The new dog's breed
    image_url (string) an image_url for the new dog

    This endpoint updates a specific dog in the database and responds with the updated version of the dog's information.

    HTTP Request

    PATCH http://localhost:9292/dogs/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the dog to update

    Request Body Parameters

    All parameters below are optional, only included parameters will update the saved values for this dog.

    Parameter Description
    name (string) The new dog's name
    birthdate (string) The new dog's birthdate in "YYYY-MM-DD" format
    breed (string) The new dog's breed
    image_url (string) an image_url for the new dog

    This endpoint creates a new dog_walk in the database and responds with the dog_walk's information in JSON format. It is used when interacting with a particular dog and logging a walk for that dog.

    HTTP Request

    POST http://localhost:9292/dog_walks

    Request Body Parameters

    These parameters will be used to create the new dog_walk.

    Parameter Description
    dog_id (integer) The ID of the dog to be walked
    walk_time (string in "YYY-MM-DDTHH:MM" format) The time of the walk to be created
    pooped (boolean) A boolean value indicating whether or not the dog pooped on this walk.

    Update a specific DogWalk

    fetch("http://localhost:9292/dog_walks/20", {
  method: "PATCH",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({

    "walk_time": "2022-05-31T11:30"
  })
})
  .then(response => response.json())

 -d '{

 http://localhost:9292/dog_walks/20

    This endpoint updates a specific dog_walk in the database and responds with the updated version of the dog_walk's information.

    HTTP Request

    PATCH http://localhost:9292/dog_walks/<ID>

    URL Parameters

    Parameter Description
    ID The ID of the dog_walk to update

    Request Body Parameters

    All parameters below are optional, only included parameters will update the saved values for this dog_walk.

    Parameter Description
    pooped (boolean) indicates whether this dog has pooped or not
    walk_time (string in "YYY-MM-DDTHH:MM" format) the updated time of the associated walk

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    404 Not Found -- The specified record could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
